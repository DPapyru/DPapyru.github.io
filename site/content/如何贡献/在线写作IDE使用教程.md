# 在线写作IDE使用教程

本文是 `/tml-ide/` 的操作参考页。

如果你只是第一次贡献，先按主线读：

1. `教学文章写作指南.md`
2. `使用网页特殊动画模块.md`

当你需要确认某个按钮在哪、某个面板怎么用，再回来查本页。

## 当前页面主要负责什么

对贡献者来说，当前网页 IDE 主要承担 4 件事：

1. 新建和编辑 Markdown 文档
2. 预览文章内容
3. 编辑 `.anim.ts` 文件并观察文章中的动画结果
4. 收集改动并统一提交

也就是说，它已经不是“只写文”的单页工具，而是“文章 + 动画 + 提交”的一个轻量工作台。

## 最常用的按钮在哪里

### 顶部工具栏

最常用的是：

1. `预览`
2. `元数据`
3. `新标签预览`
4. `统一提交`

如果当前文件是 `.fx`，还会看到 `.fx` 编译与 `渲染预览`，但这不是新人默认主线。

### 左侧文件栏

最常用的是：

1. `新增文件`
2. `重命名`
3. `删除`

新人最常用的“新增文件”会打开“快速创建”弹窗，里面已经预设了：

1. `Markdown (.md)`
2. `动画 TS (**/*.anim.ts)`
3. `代码 C# (**/code/*.cs)`
4. 图片、视频、Shader 等资源

## Markdown 写作时最常用的动作

### 1. 插入模板

如果你不知道正文怎么起步，先点 `插入模板`。

它的作用不是帮你写完文章，而是让你立刻得到一个能改的骨架，避免空白页焦虑。

### 2. 元数据

`元数据` 面板适合做两件事：

1. 核对 `title`、`description` 等字段是否完整
2. 避免把 front matter 写乱

### 3. 预览

点击 `预览` 后，当前编辑区会切换到预览/可视化编辑视图。

当前可视化编辑更适合处理：

1. 标题
2. 段落
3. 列表
4. 引用块

代码块、表格、复杂嵌入，仍然建议回源码编辑。

### 4. 新标签预览

`新标签预览` 更像“读者视角”。

适合在这些场景下使用：

1. 检查文章的完整阅读体验
2. 验证链接、嵌入、viewer 渲染结果
3. 不想在编辑区和预览区之间来回切换

### 5. 发布前自检

点击 `发布前自检` 后，IDE 会优先帮你扫常见问题，例如：

1. 占位词 `TODO / TBD / 待补充`
2. 旧语法 `{{cs:...}} / {{anim:...}} / {{ref:...}}`
3. 一些明显可疑的文档内容

它不是完整 CI，但非常适合作为提交前的第一道手工筛查。

## anim.ts 在 IDE 里怎么编辑

推荐顺序：

1. 先打开 Markdown 文章并插入 `anims:` 或 `animts` 路径。
2. 再打开对应的 `.anim.ts` 文件。
3. 修改动画文件后，回文章点 `预览` 观察结果。

当前真实行为重点记住 3 条：

1. 只有文章引用了 `*.anim.ts`，动画预览链路才会激活。
2. 只有被文章引用的 `.anim.ts` 已在工作区打开，编译与同步才会继续。
3. 当前 `.anim.ts` 补全最核心的是 `this._ctx.` 下的 `Width / Height / Time / Input`。

如果你在动画文件里看到了这些补全，说明自己已经进入了正确编辑上下文。

## 统一提交怎么用

推荐顺序：

1. 点击 `统一提交`
2. 先点 `收集改动`
3. 检查“可提交文件”里是否包含当前文章和相关资源
4. 需要时填写 PR 标题
5. 最后再点 `提交全部`

对新人来说，最重要的不是第一次就成功发 PR，而是先确认这块能正确收集到你的改动。

## 常见问题

### 1. 我新建了文件，但树里一时没找到

先确认当前活动标签已经切到新文件；如果在活动文件里能看到内容，通常说明文件已经创建成功，再回左侧资源树定位即可。

### 2. 我点了预览，但还是想改正文

直接点击预览区的“返回编辑”或使用可视化编辑旁的“跳转源码编辑”。

### 3. anim.ts 补全和普通 TS 不一样

这是正常的。当前项目更强调动画上下文里的实用补全，而不是完整通用 TypeScript 教学体验。

### 4. 什么时候才需要本地命令

默认写文流程里，不必先配本地环境。

只有在你准备提交前做更完整验证，或者 reviewer 明确要求时，再回仓库根目录执行：

```bash
npm run build
npm run check-generated
```

## 一句话总结

把 `/tml-ide/` 当成“写文章、看预览、改动画、交改动”的工作台就够了；其余能力按需再学，不必一次全背。
