DPapyru-- 语法说明

            
简化用法：一个 state slash 里用一个 make slash { ... } 完成弧线点生成，再由管线自动按运动方向取法线生成双边顶点（位置/颜色/uv），拼接三角形后应用。需要细控时依然可写 emit(...)。

            
页面内置两个 DP 预设：刀光 与 单个矩形。你可以在顶部工具栏切换预设，再点“应用脚本”或“重置示例”快速回到该预设。

            
profile swing demo {
    const int SLASH_TICKS = 24;

    state slash(SLASH_TICKS) {
        tick {
            let float t = state_tick / SLASH_TICKS;
            let float cx = 0.46;
            let float cy = 0.58;
            let float a = -2.72 + t * 2.34;

            make slash {
                center(cx, cy);              // 必填: 中心点
                radius(0.34);                // 可选: 半径
                angle(a - 0.84, a + 0.96);   // 必填: 起止弧度
                count(20);                   // 可选: 点数量
                width(1.20, 0.26);           // 可选: 起止宽度
                color(rgba(0.94, 0.94, 0.94, 0.22), rgba(1.00, 1.00, 1.00, 0.14));
                uv(0.00, 1.00);              // 可选: U 区间
            }
        }
    }

    trail {
        shader_pass 0;
        use_shader on;
        coord uv;
        uv linear;
    }
}

快速上手:
- 先选择顶部“预设”: 刀光 / 单个矩形
- 点“应用脚本”运行当前编辑器内容
- 点“重置示例”恢复当前预设源码

别名:
- biu == emit
- rainbow == rgba
- v2 == vec2
- slash/arc/swing: 兼容旧写法, 直接生成弧线点
- make slash { ... }: 推荐封装写法


            
内置方法参数（前端运行时）

            
make slash { ... } 字段与参数
- center(x, y) | at(x, y) | pos(x, y)
  类型: number, number; 必填; 表示拖尾弧线的中心点
- angle(start, end)
  类型: number, number(弧度); 必填; 定义弧线区间
- radius(r)
  类型: number; 默认 0.34; 半径
- count(n) | samples(n) | points(n)
  类型: int; 默认 20; 范围 2..96; 轨迹点数量上限
  展开: 实际显示点数会按时间渐进增长，默认公式 `visible = round(count * pow(state_progress, 3.5))`
- width(wStart, wEnd) | thickness(wStart, wEnd)
  类型: number, number; 默认 1.22 -> 0.28
- color(cStart, cEnd) | colour(cStart, cEnd)
  类型: rgba, rgba; 默认淡白到透明
- uv(uStart, uEnd) | u(uStart, uEnd)
  类型: number, number; 默认 0 -> 1

emit / biu
- emit(v2(x, y), width, rgba(r,g,b,a), u)
- emit(x, y, width, rgba(r,g,b,a), u)
- emit(x, y, width, r, g, b, a, u)

emit mesh (顶点+索引)
- emit mesh { ... }
- topology triangle_list | triangle_strip
- vertices [v(x,y,rgba,u,v), ...]
- indices [i0, i1, i2, ...] (可选)
- vertex_offset / num_vertices / index_offset / primitive_count

slash / arc / swing (兼容)
- slash(x, y, radius, start, end, widthStart, widthEnd, colorStart, colorEnd)
- slash(x, y, radius, start, end, count, widthStart, widthEnd, colorStart, colorEnd)
- slash(x, y, radius, start, end, widthStart, widthEnd, colorStart, colorEnd, uStart, uEnd)
- slash(x, y, radius, start, end, count, widthStart, widthEnd, colorStart, colorEnd, uStart, uEnd)

颜色与向量
- rgba(r, g, b, a), rainbow(r, g, b, a)
- vec2(x, y), v2(x, y)

trail 配置
- shader_pass 0..8
- use_shader on/off
- coord uv | center | auto
- uv linear | manual (point == manual)
- blend 推荐用顶部按钮切换 (脚本里的 blend 为兼容项)

状态机语义
- profile swing name { ... }: 声明一个状态机拖尾脚本
- state s(TICKS) { tick { ... } }: 每帧执行一次 tick
- goto next; / if (cond) goto next;: 状态跳转
- 若不写 goto, 状态会在时长结束后按定义顺序自动切换
- 内置时序变量: state_tick/state_time/phase_tick/tick/global_tick

默认顶点管线(前端)
1) 保存脚本输出点与运动方向
2) 用方向法线生成左右顶点
3) 顶点携带: position + color + uv
4) 相邻段拼接为三角形
5) 渲染三角形并可应用 Toy-HLSL 采样色

UV 对齐说明
- uv linear: 整条拖尾按顺序线性映射 0 -> 1
- uv manual: 使用点上的 u 值
- make slash 的 uv(...) 或 slash(..., uStart, uEnd) 在 linear 下也会触发点级 UV
- 采样区域映射到当前渲染窗口(整张画布), 仅用于刀光颜色采样

DP 背景说明
- DP 模式背景固定黑色, 用于观察刀光形状与亮度

透明底说明
- 默认背景为 16px 黑白棋盘格, 作为透明区域标识
- 顶部背景可切换: 透明棋盘 / 白色 / 黑色, 用于观察不同透明与亮度对比

混合模式说明
- alpha:
  src_rgb = shader_rgb * vertex_rgb
  a = clamp(vertex_a)
  out_rgb = src_rgb + dst_rgb * (1 - a)
  a = 1: 完全覆盖
  a = 0: 会出现 src + dst 的叠加效果
- additive:
  src_rgb = shader_rgb * vertex_rgb
  a = clamp(vertex_a)
  out_rgb = dst_rgb + src_rgb * a
  始终叠加, a 控制叠加强度

状态机内置变量
- state_tick, state_time, phase_tick, tick, global_tick
- state, phase, Time, Dt, Progress, PI



            
教程一：使用教程 / 页面说明

            
这个页面是一个实时 HLSL 预览器。你可以一边写代码，一边看右侧画布输出，并在 Console 中查看编译错误。


            
0. 适用对象与目标

            
如果你正在写 tModLoader 的特效（武器拖尾、命中闪光、UI 特效、屏幕后处理），这个页面用于快速验证公式、采样方式与多 Pass 组合。

            
目标不是直接替代项目内所有 fx 文件，而是先在这里完成“可视化调参 + 逻辑验证”，再迁移到实际模组工程。


            
1. 页面分区

            
    
                
  • 顶部工具栏：运行控制、重播（重置播放时钟）、手动编译、重置、Pass 管理、布局切换、采样越界模式（Wrap/Clamp）。
    • 
                
  • 参数调节区：iTime 实时值与 Command 变量调参。
    • 
                
  • Pass 标签区：切换当前编辑目标（Common 或某个 Pass）。
    • 
                
  • 左侧编辑区：编写 HLSL；支持自动编译。
    • 
                
  • 右侧预览区：实时画布输出与状态显示。
    • 
                
  • 底部输入区：配置 iChannel 输入、上传纹理、导出 .fx。
    • 
            


            
2. 基本使用流程

            
    
                
  1. 先在 Common 写公共函数（可选）。
    2. 
                
  2. 编辑当前 Pass 的入口函数并观察实时结果。
    3. 
                
  3. 在 Shader Inputs 里给 iChannel0-3 绑定纹理。
    4. 
                
  4. 需要多阶段渲染时新增 Pass，并在下游 Pass 读取上游 Buffer。
    5. 
                
  5. 满意后点击 生成 .fx 并复制到模组项目。
    6. 
            


            
2.1 建议调试顺序

            
    
                
  1. 先做最小可运行（只返回纯色或 uv 可视化）。
    2. 
                
  2. 再加时间项（iTime）确认动画节奏；可用参数调节区快速偏移时间。
    3. 
                
  3. 再加纹理采样（iChannel / uImage）。
    4. 
                
  4. 最后再加复杂逻辑（分支、反馈、混合）。
    5. 
            


            
3. Pass 类型说明

            
    
                
  • Buffer Pass：中间结果缓冲，常用于反馈、噪声累积、预处理。
    • 
                
  • Image Pass：最终显示层，通常作为最后输出。
    • 
            


            
3.1 常见多 Pass 结构

            
    
                
  • Pass A（Buffer）：噪声/遮罩生成。
    • 
                
  • Pass B（Buffer）：边缘检测或模糊。
    • 
                
  • Pass C（Image）：把 A/B 与主纹理合成输出。
    • 
            


            
4. 快捷键

            
    
                
  • Ctrl + Enter：手动编译。
    • 
                
  • Ctrl + S：生成 .fx。
    • 
                
  • Tab：缩进。
    • 
                
  • Shift + Tab：反缩进。
    • 
                
  • Ctrl + Space：手动触发自动补全。
    • 
            


            
5. 常见问题

            
    
                
  • 画面全黑：先检查入口函数是否返回有效 alpha，及 iChannel 是否绑定纹理。
    • 
                
  • 编译报错定位困难：先把复杂表达式拆成中间变量，一段段恢复。
    • 
                
  • UV 超出范围行为不对：确认顶部采样模式。Wrap 循环 类似 tModLoader 的 LinearWrap，Clamp 截断 类似 LinearClamp。
    • 
                
  • 效果在项目里不一致：确认项目内采样坐标系、混合状态和参数来源一致。
    • 
            


            
教程二：编写教程

            
本编辑器面向 HLSL 3.0 子集，并兼容常见 tModLoader fx 写法。推荐从“最小可运行”开始，再逐步拆成多 Pass。


            
0. 编写前 checklist

            
    
                
  • 先确定你要做的是“颜色变化”“形状变化”还是“采样重组”。
    • 
                
  • 先定义输入：时间、纹理、屏幕坐标、角色参数。
    • 
                
  • 先决定是否需要 Buffer Pass（反馈/预处理常需要）。
    • 
            


            
1. 入口函数写法

            
// 写法 A：ShaderToy 风格
float4 mainImage(float2 fragCoord)

// 写法 B：out 参数风格
void mainImage(out float4 fragColor, float2 fragCoord)

// 写法 C：tModLoader 常见像素着色器风格
float4 MainPS(float2 texCoord : TEXCOORD0) : COLOR0


            
2. 可用变量与纹理

            
float iTime, iTimeDelta;
int iFrame;
float3 iResolution;
float4 iMouse;
float4 iDate;
sampler2D iChannel0..3;
sampler2D uImage0..3;   // 自动映射到 iChannel0..3


            
2.1 坐标建议

            
// UV 约定: 左上(0,0), 右下(1,1)
// 屏幕坐标 -> 0~1
float2 uv = fragCoord / iResolution.xy;

// 中心化坐标 -> -1~1
float2 p = uv * 2.0 - 1.0;


            
2.2 自定义变量（Command）示例

            
// 标量参数：会在“参数调节”区自动出现
float speed = 1.0;       // Command(min=0,max=5,step=0.1)
float amplitude = 0.5;   // Command(0, 2, 0.1)
int iterations = 3;      // Command(min=1,max=8,step=1)

// iTime：可在参数调节区直接输入，并可自定义输入框步长
// 例如把步长改为 0.5，便于做节奏更粗的调参

// 向量/矩阵固定值（当前推荐写法）
float2 uvOffset = float2(0.0, 0.0);
float2x2 rotMatFixed = float2x2(1.0, 0.0, 0.0, 1.0);

// 向量/矩阵可变值：先拆成可调标量，再组装
float m00 = 1.0; // Command(min=-2,max=2,step=0.1)
float m01 = 0.0; // Command(min=-2,max=2,step=0.1)
float m10 = 0.0; // Command(min=-2,max=2,step=0.1)
float m11 = 1.0; // Command(min=-2,max=2,step=0.1)
float2x2 rotMat = float2x2(m00, m01, m10, m11);


            
3. 常用类型与函数

            
// 类型
float2/3/4, int2/3/4, uint2/3/4, bool2/3/4
float2x2, float3x3, float4x4
half/fixed/min16float

// 函数映射
lerp, frac, rsqrt, ddx, ddy, saturate
tex2D, tex2Dproj, tex2Dgrad, tex2Dbias
mul, clip, atan2, mad, rcp, log10


            
4. 编写建议（tModLoader 习惯）

            
    
                
  • 在 Common 放工具函数，Pass 内只写当前阶段逻辑。
    • 
                
  • Image Pass 尽量只负责最终合成，避免塞入过多状态逻辑。
    • 
                
  • 需要移植到项目时，优先保留函数结构与参数命名，再替换业务纹理与参数。
    • 
            


            
5. 最小示例到可用示例

            
// Step 1: 最小可运行
float4 MainPS(float2 texCoord : TEXCOORD0) : COLOR0 {
    // UV 约定: 左上(0,0), 右下(1,1)
    float2 uv = texCoord;
    return float4(uv, 0.0, 1.0);
}

// Step 2: 增加时间动画
float4 MainPS(float2 texCoord : TEXCOORD0) : COLOR0 {
    float2 uv = texCoord;
    float wave = 0.5 + 0.5 * sin(iTime + uv.x * 8.0);
    return float4(uv.x, uv.y, wave, 1.0);
}


            
6. 迁移到 tModLoader 的建议步骤

            
    
                
  1. 先把入口函数与工具函数迁移到项目 fx。
    2. 
                
  2. 把 iChannel/uImage 替换成项目里的实际 sampler。
    3. 
                
  3. 把 iTime/iResolution 替换为项目参数并在 C# 侧写入。
    4. 
                
  4. 最后在游戏中验证混合状态与绘制顺序。
    5.
