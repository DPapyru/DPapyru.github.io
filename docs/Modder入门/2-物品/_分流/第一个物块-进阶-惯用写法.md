这份补充内容面向 **C≥1 且 T≥1**：默认你已能阅读本章代码，因此这里聚焦更惯用、可维护性更好的组织方式与注意点。

## 1）把"可调参数"集中起来

物块的属性往往需要反复调整。建议将其集中为常量：

```csharp
private const int MinPickRequired = 40;  // 最低镐力
private const float MineResist = 2f;  // 挖掘阻力
```

## 2）`SetStaticDefaults()` 里只做"静态属性"

保持 `SetStaticDefaults()` 的目标单一：只维护物块的静态属性，避免堆叠复杂逻辑。

## 3）物块物品的分离设计

在 tModLoader 中，物块和物块物品通常是分离的：

- `ModTile` 类：定义物块在世界中的行为
- `ModItem` 类：定义物品在背包中的行为

这种设计让代码更清晰，便于维护。
